OpenClaw Gateway — SSH Tunnel Access

Why tunnel-only?

The OpenClaw Gateway runs on 127.0.0.1:18789 and is not exposed publicly. This avoids adding another attack surface to the EC2 instance. Access is via SSH tunnel only — your browser connects to localhost:18789 which forwards through the encrypted SSH session to the server.

Step 1 — Open a terminal on your laptop

Any terminal works: Terminal.app, iTerm2, Windows Terminal, etc.

Step 2 — Run this command

ssh -i ~/.ssh/cowork-key.pem -L 18789:127.0.0.1:18789 ubuntu@3.84.52.62

Step 3 — Open the Gateway

Once the tunnel is up, open http://localhost:18789 in your browser.

Troubleshooting

ProblemFix
Permission denied chmod 600 ~/.ssh/cowork-key.pem
Connection refused Confirm the gateway service is running — check the Health tab.
Port already in use Kill the stale tunnel: lsof -ti:18789 | xargs kill -9

Tunnel closes when you Ctrl-C the SSH session.
